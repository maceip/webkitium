## What changed during this audit

### 0009 — regenerated

Original used zero-context hunk form (`@@ -32,0 +33 @@`) which `git apply`
rejects without `--unidiff-zero`. Regenerated with proper context hunks
by applying + `git diff`. Content-identical to the original; just a
format normalization so every patch applies with the same command.

### 0033 — rewritten

Original (ported from WebKit-ng) matched an intermediate upstream state
we don't have. Regenerated from scratch by running the intended edit
against the pin and capturing `git diff`. Produces the same end result:
`backendType = Undefined`, failure-path logger added.

### 0034 — merged into existing `WebGPUDawnCompat.h/.cpp`

The initial draft created new files `WebGPUDawnCompatWindows.{h,cpp}`.
That turned out to conflict with patch 0005 (pre-existing), which already
creates `Source/WebCore/Modules/WebGPU/Implementation/WebGPUDawnCompat.{h,cpp}`
— same namespace (`WebCore::WebGPU::DawnCompat`), same `#if PLATFORM(WIN)`
gate, and its `.cpp` is already registered in `Sources.txt`. The surface
helpers were appended to that existing file instead:

- `WGPUSurface createSurfaceForHWND(WGPUInstance, void* hwnd, void* hinstance, const char* label)`
- `bool configureSurface(WGPUSurface, WGPUDevice, WGPUTextureFormat, WGPUTextureUsage, uint32_t, uint32_t)`

### 0035 — rewritten

First draft had the wrong path (`Modules/WebGPU/Implementation/` instead
of the correct `html/canvas/`) and two FIXME placeholders. Rewritten at
`Source/WebCore/html/canvas/GPUCanvasContextWin.{h,cpp}`, following the
Cocoa sibling's layout, with no placeholders.

### 0037 / 0038 / 0039 — rewritten

Earlier drafts had guessed line numbers / context lines against upstream
main, which differs from our pin. Regenerated by applying the intended
edits to the post-0033-cumulative pin tree and capturing `git diff`.

### 0040 — simplified

First draft registered `WebGPUDawnCompatWindows.cpp` in PlatformWin.cmake.
After 0034 was merged into the existing `WebGPUDawnCompat.cpp` (already
registered by 0005), this patch now only needs to register
`html/canvas/GPUCanvasContextWin.cpp` + the private framework header.

## Standing rule

Every patch added to `webkit/patches/windows/` must pass a cumulative
`git apply --check -p1` run against the pin + all prior patches. The
verification script lives at
`changes/windows-webgpu-service/harness/tools/verify-patches.sh`
(to be added) so the check can be rerun on any machine with the
tarball available.
